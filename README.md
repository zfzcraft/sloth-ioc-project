## Sloth IOC
A **cloud-native, zero-scan, ultra-lazy IOC container** designed for Kubernetes & microservices.

### Features
- No BeanDefinition layer
- Zero classpath scanning at runtime
- Constant startup time (300~500ms)
- Startup speed **independent** of business bean count
- Ultra lazy loading: no usage → no class load, no instance create
- Powered by compile-time index & Maven plugin
- Lightweight, low memory, stable & predictable
- Full compatible with MVC, AOP, extension plugins

### Philosophy
Lazy to the extreme, fast to the limit.
Back to the essence of IOC.

## Sloth IOC
专为 Kubernetes 与微服务打造的**云原生零扫描极致懒加载 IOC 容器**

### 核心特性
- 彻底剔除 BeanDefinition 层
- 运行时零类路径扫描
- 恒定启动耗时（300~500ms）
- 启动速度与业务 Bean 数量完全无关
- 极致懒加载：未被使用 → 不加载类、不创建实例
- 基于编译期索引与 Maven 插件驱动
- 轻量低内存、稳定可预期
- 全面兼容 MVC、AOP 与扩展插件体系

### 设计理念
懒至极致，快至巅峰。
回归 IOC 本质。
